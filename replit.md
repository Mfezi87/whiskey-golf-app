# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.

## App Features (Whiskey Golf)

### Implemented
- **Auth**: Session-based (express-session + connect-pg-simple), 7-day expiry, username/password
- **Tournaments**: Create, view, edit, delete tournaments with configurable salary cap, roster, scoring
- **Tournament Access**: Commissioner system, visibility (public/private), join mode (open_join/approval_required/invite_only/link_only), invite link generation
- **Participants**: Full state machine (invited → joined/rejected, requested → joined/rejected, joined → left/removed). Commissioner can invite by username, approve/reject join requests, remove members.
- **Draft**: Snake or alternate pick order, salary cap, captain designation, manage mode for commissioners
- **Results**: Manual results entry (finish position, birdies, eagles, bogeys, missed cut), replacement golfer support
- **Scores**: Auto-computed fantasy scores with position points, captain multiplier, bonus/penalty points
- **Leaderboard**: All-time standings, head-to-head rivalry stats

### DB Tables
- `users` — username, display_name, password_hash
- `sessions` — connect-pg-simple session storage
- `tournaments` — all tournament metadata + commissioner_user_id, visibility, join_mode, invite_link_token, invite_link_enabled
- `tournament_configs` — per-tournament settings
- `tournament_position_points` — finish position → points mapping
- `tournament_participants` — participant state machine (invited/requested/joined/rejected/removed/left)
- `tournament_golfers` — golfer pool per tournament (uploaded via CSV)
- `fantasy_teams` — one per user per tournament
- `draft_picks` — picked golfers with captain flag
- `golfer_results` — actual results for scoring
- `replacement_results` — replacement golfer results

### Key Backend Notes
- `commissionerUserId` is nullable — legacy tournaments created before this feature have NULL
- `lib/api-zod/src/participants.ts` — hand-written Zod contracts (not generated by Orval)
- `lib/api-zod/src/generated/` — never touch these (Orval-generated)
- Session `sameSite: "lax"`, `secure: isProduction`, `trust proxy: 1` required for Replit
- `useToast` import: `@/hooks/use-toast` (not `@/components/ui/use-toast`)
